Guía Rápida de Despliegue Local - SCI-TSS

Estimado Ingeniero, a continuación los pasos precisos para levantar la aplicación SCI-TSS en su entorno local (Apache/MySQL) para revisión.

PASO 1: Ubicación de Archivos

Descomprima el archivo ZIP recibido.

Copie la carpeta completa del proyecto (ej. php-app) dentro del directorio público de su servidor local:

Si usa XAMPP: C:\xampp\htdocs\

Si usa WAMP: C:\wamp64\www\

PASO 2: Configuración de la Base de Datos

Abra el panel de control de XAMPP/WAMP y asegúrese de que Apache y MySQL estén en ejecución.

Ingrese a phpMyAdmin en su navegador: http://localhost/phpmyadmin

Vaya a la pestaña Bases de datos y cree una nueva base de datos llamada: carnetizacion_tss (Seleccione el cotejamiento utf8mb4_unicode_ci).

Haga clic sobre la base de datos carnetizacion_tss recién creada, vaya a la pestaña Importar.

Importe los archivos SQL en el siguiente orden OBLIGATORIO:
1. db/00_tablas_espanol.sql  ← PRIMERO: Crea las tablas en español (usuarios, empleados, auditoria_logs)
2. db/01_schema_produccion_final.sql  ← RBAC, auditoría y tablas de soporte
3. db/02_consolidacion_spanish.sql  ← Migra permisos y roles al esquema español
4. db/fix_indexes.sql  ← Índices de rendimiento (idempotente)

El paso 1 es crítico: sin él el sistema falla al autenticar. Haga clic en Continuar / Importar tras cada archivo.

PASO 3: Configuración de Variables de Entorno (.env)

Vaya a la carpeta raíz del proyecto (htdocs/php-app/).

Busque el archivo llamado .env.example.

Cópielo y renombre la copia para que se llame exactamente .env (sin ninguna extensión adicional).

Ábralo con cualquier editor de texto y verifique que los datos de MySQL correspondan a su entorno local (por defecto en XAMPP es root sin clave):

DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=carnetizacion_tss
DB_USER=root
DB_PASS=


PASO 4: Ingreso al Sistema

Abra su navegador e ingrese a la ruta local del proyecto: http://localhost/php-app/

El sistema lo redirigirá al Login. Utilice las credenciales de Administrador pre-cargadas en el archivo SQL:

Usuario: admin

Contraseña: Admin123!

Nota de Seguridad (T8): Al ser el primer ingreso, la arquitectura de seguridad forzará automáticamente una pantalla de cambio de contraseña obligatoria. Proceda a cambiarla para continuar al Dashboard.

¡El sistema ya está completamente operativo en su equipo!
